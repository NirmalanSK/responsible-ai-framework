RULE 1 — OBSERVATION INSERTION / DELETION
P(Y | do(X), Z, W) = P(Y | do(X), W)
if (Y ⊥ Z | X, W) in mutilated graph GX̄
Tamil: Z என்ற variable Y-க்கு irrelevant-ஆ இருக்கும்போது remove பண்ணலாம். உங்க framework: Amazon case-ல் shoe size, hair color போன்ற irrelevant variables graph-இலிருந்து remove → TCE calculation faster + accurate.
RULE 2 — ACTION / OBSERVATION EXCHANGE ← MOST USED
P(Y | do(X), Z) = P(Y | X, Z)
if (Y ⊥ Z | X) in GX̄ — no backdoor path
Tamil: Backdoor path இல்லாதபோது do() → observe() replace பண்ணலாம். உங்க TCE formula:

RULE 3 — ACTION INSERTION / DELETION
P(Y | do(X), do(Z), W) = P(Y | do(X), W)
if (Y ⊥ Z | X, W) in GX̄Z̲
Tamil: X-இலிருந்து Z-க்கு causal path இல்லாதபோது do(Z) remove பண்ணலாம். Tay case: do(block_repeat) → do(reward_framing) automatically ineffective. Multiple interventions-ல் எது necessary என்று identify பண்றது.
